Database Relationships

Understanding relationships between tables is crucial for designing efficient databases. Learn about different types of relationships and how to implement them.

Types of Relationships

One-to-One

Users
Profiles

Each record in Table A relates to one record in Table B

-- One user has one profile
Users: id, name, email
Profiles: id, user_id, bio, avatar

One-to-Many

Customers
Orders

One record in Table A relates to many records in Table B

-- One customer has many orders
Customers: id, name, email
Orders: id, customer_id, amount, date

Many-to-Many

Students
Enrollments
Courses

Many records in Table A relate to many records in Table B

-- Students enroll in many courses
Students: id, name
Courses: id, title
Enrollments: student_id, course_id, grade

Foreign Keys and Constraints

CREATE TABLE customers (
    id INT PRIMARY KEY AUTO_INCREMENT,
    name VARCHAR(100) NOT NULL,
    email VARCHAR(100) UNIQUE
);

CREATE TABLE orders (
    id INT PRIMARY KEY AUTO_INCREMENT,
    customer_id INT,
    amount DECIMAL(10,2),
    order_date DATE,
    FOREIGN KEY (customer_id) 
        REFERENCES customers(id)
        ON DELETE CASCADE
        ON UPDATE CASCADE
);

Practice Exercise

Scenario: Design a database for a library system with the following requirements:

  • Books have title, author, ISBN, and publication year
  • Members have name, email, and membership date
  • A member can borrow multiple books
  • A book can be borrowed by multiple members (at different times)
  • Track borrowing date and return date

You'll need 3 tables: books, members, and borrowings. The borrowings table should be a junction table with foreign keys to both books and members.
